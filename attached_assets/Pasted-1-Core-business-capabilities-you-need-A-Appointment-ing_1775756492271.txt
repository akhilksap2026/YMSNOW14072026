B. Driver pre-arrival / PreCheck

Vector’s PreCheck sends an SMS link before arrival, lets the driver complete check-in ahead of time, and validates driver identity using submitted driver name, CDL last 4, DOT, and carrier details. It also supports GPS ETA and temperature input for reefer loads.

What to build

Architecture note
Use a short-lived signed check-in token rather than login-required flow for drivers. Drivers are external users; friction kills adoption.

C. Multi-mode check-in UX

Vector supports guard-initiated, driver-initiated, and kiosk-initiated check-ins, all backed by mobile, SMS/mobile web, or desktop flows.

What to build

Driver-initiated
SMS link / QR code

Search appointment / scan QR / lookup phone
Assisted completion for walk-ins or exceptions
Kiosk-initiated

Key principle
Do not build separate workflows per channel. Build:

a shared CheckIn Workflow Engine
a shared Task Definition Engine
channel-specific UIs only

That prevents rule duplication.

Most YMS implementations fail here by conflating appointment, load, and visit. Keep them separate:

Appointment = planned time window
Load/Shipment = business movement
Visit = physical trip instance at the facility

That separation is essential for reschedules, walk-ins, drops, swaps, and re-check-ins.

6) Non-obvious design decisions
Use workflow state machines, not just CRUD

Digital check-in is a process, not a form. Use a finite-state machine or workflow engine for visit lifecycle.

Make everything facility-configurable

Different sites will require different tasks, identity checks, gates, and dock assignment rules.

Prefer event-driven integration over synchronous coupling

Gate operations, messages, dock assignment, and external system updates should be decoupled by events.

External-user UX matters more than internal elegance

Drivers should be able to complete most flows with:

SMS
browser
no password
minimal typing
multilingual prompts
Support both appointment-backed and walk-in visits

Not every visit will have clean upstream planning data.
